# 不吃灰桌面版布局校准设计

## 目标

让桌面版在常见的 1920×1080 浏览器视口中无需滚动即可完整展示，视觉比例参考用户提供的图1。移除左上品牌和左侧宣传文案，只保留右上导航与居中的游戏机主体。

## 页面结构

- 页面背景继续使用现有蓝色到白色渐变。
- 右上角仅保留“首页 / 收藏库 / 关于”。
- 游戏机主体水平居中，整体高度受视口高度约束，完整显示屏幕、控制面板、紫色柜子和三张卡带。
- 本轮仅校准桌面版；保留现有移动端规则，但不继续细化。

## 主机素材与屏幕分层

- 使用 `组件/computer2.png` 作为主机结构和黑屏状态的校准基准。
- 将 `Clipboard - 2026-07-18 18.15.34.png` 复制为站内待机壁纸素材。
- 屏幕内容层固定为 16:9，边界严格落在主机屏幕内框之内。
- 待机状态显示完整 16:9 壁纸，采用 `object-fit: cover` 或精确裁切，不能出现黑屏越过屏幕下边界的情况。
- 关机状态显示黑色 16:9 屏幕和白色像素笑脸。
- 读盘与模拟播放继续复用同一个 16:9 内容层，不改现有状态机。

## 控制面板定位

- Power 按钮位于白色控制面板左侧，按钮与 `POWER` 标签形成一个独立定位组。
- 插槽保持在控制面板水平中心。
- 全屏按钮位于白色控制面板右侧，按钮与“全屏”标签形成独立定位组。
- 两个按钮都不能压到紫色柜子，也不能越过白色控制面板上下边界。
- 定位统一相对于主机容器计算，避免相对页面或卡带柜产生漂移。

## 右上导航

- 移除当前蓝色方块按钮视觉。
- 使用用户图5的白色填充、黑色像素描边图标和中文文字。
- 导航靠页面右侧布局，并与顶部保留安全间距。
- 保留现有点击“页面准备中”反馈，不实现真实跳转。

## 卡带与交互

- 保留绿色、蓝色、粉色三张 HTML/CSS 矢量卡带及其文字内容。
- 卡带随主机整体等比缩小，仍叠放在紫色柜子上方。
- 桌面端悬停或键盘聚焦卡带时，卡带整体向上移动约 8–10px，并适度增强像素阴影，表达“已选中”。
- 移除卡带悬停与聚焦状态现有的白色虚线边框，不使用额外外框包围卡带。
- 保留点击播放、桌面拖放、读盘、Power、全屏、Esc/关闭按钮和导航反馈。
- 本轮不调整数据结构和第三方链接。

## 验收标准

- 1920×1080 与 1440×900 桌面视口中，页面主体无需纵向滚动即可完整显示。
- 屏幕内容区域为 16:9，待机壁纸、黑屏、读盘和播放器都不越过主机屏幕边框。
- Power 和全屏按钮完整位于白色控制面板中。
- 右上导航与图5保持相同的黑白像素描边方向，并明显靠右。
- 卡带悬停时平稳上移且没有白色虚线选中框，卡带之间和控制面板之间不发生遮挡。
- 现有交互测试、构建检查全部通过。
